ProgressBar

프로그레스 바는 작업 진행 상황, 로딩 프로세스, 완료율 등을 시각적으로 표시하기 위한 컴포넌트입니다. 사용자에게 진행 중인 작업의 완료 정도를 백분율로 보여줍니다.

인터랙티브 예제​

아래 UI에서 다양한 속성을 직접 변경하여 ProgressBar 컴포넌트의 모습을 확인해보세요.

50%
valuenumber

labelselect
진행 상태 라벨

기본 사용법​

import { ProgressBar } from '@ncds/ui';

function App() {
  return <ProgressBar value={50} />;
}

기본 프로그레스 바​

가장 기본적인 형태의 프로그레스 바입니다. value prop만 설정하면 됩니다.

다양한 레이블 위치​

프로그레스 바는 4가지 다른 레이블 위치를 지원합니다.

85%

다양한 값으로 진행 정도 표시​

프로그레스 바는 0에서 100 사이의 값을 표시합니다. 값이 범위를 벗어나면 자동으로 0과 100 사이로 조정됩니다.

valueToPercent 옵션 사용​

valueToPercent 옵션은 segments가 2개 이상일 때만 적용됩니다. 기본값은 true이며, 각 segment의 합계를 100%로 규정하고 비율로 계산합니다.

기본 동작 (valueToPercent=true, 기본값)​

기본적으로 각 segment의 합계를 100%로 규정하고 비율로 계산합니다.

50%50%
50%50%

위 예제는 합계 60을 100%로 규정하여 각각 50%씩 표시합니다 (30/60 * 100 = 50%).

valueToPercent=false로 설정​

valueToPercent={false}를 명시적으로 설정하면 각 segment의 value가 퍼센트로 간주됩니다.

30%30%
30%30%

위 예제는 30% + 30% = 60%를 표시합니다.

다양한 비율 예제​

17%33%50%
25%25%25%25%
17%33%50%
25%25%25%25%

참고: valueToPercent는 segment가 1개이거나 value prop만 사용할 때는 무시됩니다.

100% 초과 값 처리​

프로그레스 바는 100%를 초과하는 값을 자동으로 제한합니다. 각 케이스별 동작을 확인해보세요.

segment가 1개일 때 100% 초과​

segment가 1개일 때는 값이 100%를 초과하면 자동으로 100%로 제한됩니다.

위 예제에서 150%와 200%는 모두 100%로 제한되어 표시됩니다.

segments가 2개 이상일 때 합이 100% 초과 (valueToPercent=false)​

valueToPercent={false}로 설정하면 각 segment의 값을 퍼센트로 사용하며, 합계가 100%를 초과해도 그대로 나타냅니다.

50%50%
60%60%
40%50%30%
50%50%
60%60%
40%50%30%

위 예제에서:

  • 첫 번째: 50% + 50% = 100% (정상)
  • 두 번째: 60% + 60% = 120% ( 막대 길이 비율 유지: 50% + 50%)
  • 세 번째: 40% + 50% + 30% = 120% (막대 길이 비율 유지: 약 33.3% + 41.7% + 25%)

segments가 2개 이상일 때 합이 100% 초과 (valueToPercent=true)​

기본적으로 valueToPercent={true}이므로 각 segment의 합계를 100%로 규정하며, 합계가 얼마든지 항상 100%로 표시됩니다.

50%50%
50%50%
33%42%25%
17%33%50%
50%50%
50%50%
33%42%25%
17%33%50%

위 예제에서:

  • 첫 번째: 합계 60 → 각각 50%씩 표시
  • 두 번째: 합계 120 → 각각 50%씩 표시 (60/120 = 50%)
  • 세 번째: 합계 120 → 각각 33.3%, 41.7%, 25% 표시 (비율 유지)
  • 네 번째: 합계 600 → 각각 16.7%, 33.3%, 50% 표시 (100/600, 200/600, 300/600)

차이점: valueToPercent={true}일 때는 합계가 얼마든지 항상 100%로 표시되며, 각 segment는 합계 대비 비율로 계산됩니다.

0%일 때 라벨 표시 제어​

기본적으로 0%일 때는 라벨이 표시되지 않습니다. showZeroLabel prop을 true로 설정하면 0%일 때도 라벨을 표시할 수 있습니다.

여러 구간으로 나눈 프로그레스 바 (Segments)​

프로그레스 바를 여러 구간으로 나누고 각 구간에 다른 색상을 지정할 수 있습니다.

1개 구간​

1개 구간일 때는 기존 value prop과 동일하게 동작합니다. 색상은 Colors 메뉴에 정의된 색상 이름을 사용할 수 있으며, segment는 400 톤, 글자는 600 톤으로 자동 적용됩니다.

2개 구간​

2개 구간일 때는 왼쪽 끝과 오른쪽 끝에 각각 퍼센트가 표시됩니다. 각 segment의 색상에 맞춰 글자 색상도 자동으로 적용됩니다.

38%63%
38%63%

3개 이상 구간​

3개 이상일 때는 각 구간의 위치에 맞춰 하단에 퍼센트가 표시됩니다. 각 segment의 색상에 맞춰 글자 색상도 자동으로 적용됩니다. 개수 제한 없이 여러 구간을 표시할 수 있습니다.

31%44%25%
31%44%25%

다양한 색상을 사용한 예제입니다.

27%33%20%13%7%
27%33%20%13%7%

색상 없이 사용하기​

색상을 지정하지 않으면 기본 색상(gray)이 사용됩니다.

33%44%22%
33%44%22%

지원되는 색상​

Colors 메뉴에 정의된 색상 중 흰색, 검은색을 제외한 색상들을 사용할 수 있습니다: red, green, blue, pink, orange, violet, gray

  • Segment 색상: 자동으로 400 톤 적용 (예: red → var(--primary-red-400))
  • 글자 색상: 자동으로 600 톤 적용 (예: red → var(--primary-red-600))

33%44%22%

HTML 사용법​

HTML에서 직접 사용할 수 있습니다. ncua.ProgressBar 클래스를 사용하여 프로그레스 바를 생성하고 DOM에 추가합니다.

기본 사용법​

<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="UTF-8" />
    <title>ProgressBar Example</title>
    <link rel="stylesheet" href="path/to/ui-admin.css" />
  </head>
  <body>
    <div id="progress-container"></div>

    <script src="path/to/ui-admin.bundle.js"></script>
    <script>
      // 기본 프로그레스 바 생성
      const progressBar = ProgressBar.create({ value: 75, label: 'right' });
      document.getElementById('progress-container').appendChild(progressBar.getElement());
    </script>
  </body>
</html>

Segments를 사용한 프로그레스 바​

<script>
  // 여러 구간으로 나눈 프로그레스 바
  const segmentsProgressBar = ProgressBar.create({
    segments: [
      { value: 30, color: 'red' },
      { value: 50, color: 'green' },
      { value: 20, color: 'blue' },
    ],
  });
  document.getElementById('progress-container').appendChild(segmentsProgressBar.getElement());
</script>
