RangeDatePickerWithButtons

RangeDatePickerWithButtons 컴포넌트는 날짜 범위를 선택할 수 있는 RangeDatePicker와 기간 버튼이 결합된 컴포넌트입니다.
사용자는 직접 날짜를 선택하거나, 오늘/1주일/1개월 등 미리 정의된 기간 버튼을 클릭하여 손쉽게 날짜 범위를 설정할 수 있습니다.

기본 사용법​

import { RangeDatePickerWithButtons } from '@ncds/ui-admin';
import { useState } from 'react';

function App() {
  const [currentButtonId, setCurrentButtonId] = useState<'NONE' | 'TODAY' | 'WEEK' | '1MONTHS' | '1YEAR' | 'ENTIRE'>(
    'TODAY'
  );
  const [startDate, setStartDate] = useState('2025-01-01');
  const [endDate, setEndDate] = useState('2025-01-31');

  return (
    <RangeDatePickerWithButtons
      currentButtonId={currentButtonId}
      setCurrentButtonId={setCurrentButtonId}
      periodKeys={['TODAY', '1WEEK', '1MONTHS', '1YEAR', 'ENTIRE']}
      startDateOptions={{
        currentDate: startDate,
        onChangeDate: setStartDate,
      }}
      endDateOptions={{
        currentDate: endDate,
        onChangeDate: setEndDate,
      }}
    />
  );
}

startDateOptionsDatePickerProps필수

시작일 DatePicker의 설정 옵션. DatePicker 컴포넌트의 모든 props를 사용할 수 있습니다.
- currentDate: string (필수) - 현재 선택된 날짜
- onChangeDate: (date: string) => void (필수) - 날짜 변경 콜백
- size: 'xs' | 'sm' (옵셔널, RangeDatePickerWithButtons의 size prop으로 덮어씌워짐)
- shouldFocus: boolean (옵셔널, 기본값: true)
- disabled: boolean (옵셔널, 기본값: false)
- autocomplete: 'on' | 'off' (옵셔널, 기본값: 'off')
- datePickerOptions: Options (옵셔널) - flatpickr 옵션
- onValidationError: (error: ValidationError) => void (옵셔널) - minDate/maxDate 범위를 벗어난 날짜에 대한 유효성 검사 오류 콜백
- destructive: string (옵셔널) - 오류 메시지
- placeholder: string (옵셔널) - placeholder 텍스트
- isEndDate는 자동으로 false로 설정됩니다.


참고: 날짜를 직접 선택하면 currentButtonId가 'NONE'으로 자동 변경됩니다.

endDateOptionsDatePickerProps필수

종료일 DatePicker의 설정 옵션. DatePicker 컴포넌트의 모든 props를 사용할 수 있습니다.
- currentDate: string (필수) - 현재 선택된 날짜
- onChangeDate: (date: string) => void (필수) - 날짜 변경 콜백
- size: 'xs' | 'sm' (옵셔널, RangeDatePickerWithButtons의 size prop으로 덮어씌워짐)
- shouldFocus: boolean (옵셔널, 기본값: true)
- disabled: boolean (옵셔널, 기본값: false)
- autocomplete: 'on' | 'off' (옵셔널, 기본값: 'off')
- datePickerOptions: Options (옵셔널) - flatpickr 옵션
- onValidationError: (error: ValidationError) => void (옵셔널) - minDate/maxDate 범위를 벗어난 날짜에 대한 유효성 검사 오류 콜백
- destructive: string (옵셔널) - 오류 메시지
- placeholder: string (옵셔널) - placeholder 텍스트
- isEndDate는 자동으로 true로 설정됩니다.


참고: 날짜를 직접 선택하면 currentButtonId가 'NONE'으로 자동 변경됩니다.

validationOption{ initDate?: { startDate?: string; endDate?: string; }; setting?: { unit: unitOfTime.Diff; period: number; } }No

날짜 유효성 및 기간 제한 옵션. RangeDatePicker의 validationOption과 동일합니다.
- initDate: 초기 날짜 설정 (옵셔널)
- startDate: 시작일 초기값
- endDate: 종료일 초기값
- setting: 기간 제한 설정 (옵셔널)
- unit: 기간 단위 ('days', 'weeks', 'months', 'years' 등 moment의 unitOfTime.Diff 타입)
- period: 기간 값 (숫자)


예: { setting: { unit: 'months', period: 1 } } - 최대 1개월 이내만 선택 가능

onDateValidation(params: { type: "start" | "end"; errorType: "reset" | "period" | "overlap" | null; newDate: string; currentDate: string; }) => voidNo

날짜 유효성 검사 시 호출되는 콜백 함수. RangeDatePicker의 onDateValidation과 동일합니다.


params 객체는 다음과 같은 속성을 가집니다:
- type: 유효성 검사가 실행된 날짜 선택기의 종류 ('start': 시작일, 'end': 종료일)
- errorType: 유효성 검사 결과 유형
- 'period': 기간 제한 초과 (validationOption.setting에서 설정한 기간을 초과)
- 'overlap': 날짜 겹침 오류 (시작일이 종료일보다 늦거나, 종료일이 시작일보다 빠름)
- 'reset': 날짜가 초기화됨
- null: 유효성 검사 통과
- newDate: 새로 선택한 날짜 (오류 발생 시 자동 조정된 날짜)
- currentDate: 현재 설정된 날짜 (오류 발생 전 날짜)

size'xs' | 'sm'No

컴포넌트 사이즈. 기본값은 'xs'입니다. ButtonGroup과 RangeDatePicker 모두에 동일하게 적용됩니다.


startDateOptions와 endDateOptions의 size prop을 덮어씁니다.

참고사항​

  • endDateOptions에 전달된 DatePicker는 자동으로 isEndDate={true}가 적용됩니다.
  • size prop은 ButtonGroup과 두 DatePicker 모두에 동일하게 적용됩니다.
  • 날짜를 직접 선택하면 currentButtonId가 자동으로 'NONE'으로 변경됩니다.
  • 기간 버튼을 클릭하면 해당 기간에 맞는 시작일과 종료일이 자동으로 설정됩니다.
  • 'ENTIRE' 버튼을 클릭하면 시작일과 종료일이 모두 빈 문자열('')로 설정됩니다.

종료일을 어제 날짜로 설정하기​

  • 종료일을 어제 날짜로 설정할 수 있습니다.
import { RangeDatePickerWithButtons } from '@ncds/ui-admin';
import { useState } from 'react';

function App() {
  const [currentButtonId, setCurrentButtonId] = useState<'NONE' | 'TODAY' | 'WEEK' | '1MONTHS' | '1YEAR' | 'ENTIRE'>(
    'TODAY'
  );
  const [startDate, setStartDate] = useState('2025-01-01');
  const [endDate, setEndDate] = useState('2025-01-31');

  return (
    <RangeDatePickerWithButtons
      useYesterdayAsEndDate={true}
      currentButtonId={currentButtonId}
      setCurrentButtonId={setCurrentButtonId}
      periodKeys={['TODAY', '1WEEK', '1MONTHS', '1YEAR', 'ENTIRE']}
      startDateOptions={{
        currentDate: startDate,
        onChangeDate: setStartDate,
        datePickerOptions: {
          minDate,
          maxDate: endDate,
        },
      }}
      endDateOptions={{
        currentDate: endDate,
        onChangeDate: setEndDate,
      }}
    />
  );
}

사용 가능한 periodKeys 값들:

  • TODAY: 오늘
  • YESTERDAY: 어제
  • 3DAYS: 3일
  • 7DAYS: 7일
  • 1WEEK: 1주일

  • 1MONTHS: 1개월
  • 3MONTHS: 3개월
  • 1YEAR: 1년
  • ENTIRE: 전체 기간
  • NONE: 버튼 선택 해제 상태

위 예시에서:

  • 날짜를 직접 선택하면 currentButtonId가 'NONE'으로 변경되어 버튼 선택이 해제됩니다.

periodItems prop을 사용하면 기본 PERIOD_ITEM 대신 프로젝트에 맞는 커스텀 기간 항목을 정의할 수 있습니다.

import { RangeDatePickerWithButtons } from '@ncds/ui-admin';
import { useState } from 'react';

const CUSTOM_PERIOD_ITEM = {
  TODAY: { text: '오늘', period: 0, unit: 'days' },
  '7DAYS': { text: '7일', period: 6, unit: 'days' },
  '1MONTHS': { text: '한 달', period: 1, unit: 'months' },
  '6MONTHS': { text: '6개월', period: 6, unit: 'months' },
  ENTIRE: { text: '전체', period: 0, unit: null },
  NONE: { text: '', period: 0, unit: null },
};

function App() {
  const [currentButtonId, setCurrentButtonId] = useState<keyof typeof CUSTOM_PERIOD_ITEM>('TODAY');
  const [startDate, setStartDate] = useState('2026-02-24');
  const [endDate, setEndDate] = useState('2026-02-24');

  return (
    <RangeDatePickerWithButtons
      periodItems={CUSTOM_PERIOD_ITEM}
      currentButtonId={currentButtonId}
      setCurrentButtonId={setCurrentButtonId}
      periodKeys={['TODAY', '7DAYS', '1MONTHS', '6MONTHS', 'ENTIRE']}
      startDateOptions={{
        currentDate: startDate,
        onChangeDate: setStartDate,
      }}
      endDateOptions={{
        currentDate: endDate,
        onChangeDate: setEndDate,
      }}
    />
  );
}

onDateValidation 콜백​

유효성 검사 결과를 받아 처리할 수 있는 콜백 함수입니다. RangeDatePicker의 onDateValidation과 동일한 인터페이스를 사용합니다.

onDateValidation={({ type, errorType, newDate, currentDate }) => {
  if (errorType === 'period') {
    console.log('기간 제한 초과:', newDate);
  } else if (errorType === 'overlap') {
    console.log('날짜 겹침 오류:', type, newDate);
  } else if (errorType === 'reset') {
    console.log('날짜 초기화됨');
  } else if (!errorType) {
    console.log('유효성 검사 통과');
  }
}}

onDateValidation vs onValidationError 차이점​

onDateValidation과 onValidationError는 서로 다른 목적과 레벨에서 동작하는 콜백 함수입니다. 두 콜백을 함께 사용할 수 있으며, 각각 다른 유형의 검증을 처리합니다.

onDateValidation (RangeDatePicker 레벨)

  • 용도: 시작일과 종료일 간의 관계 및 기간 제한 검증
  • 검증 항목:
    • overlap: 시작일이 종료일보다 늦거나, 종료일이 시작일보다 빠름
    • period: validationOption.setting에서 설정한 기간 제한 초과 (예: 최대 1개월)
    • reset: 날짜 초기화
    • null: 유효성 검사 통과
  • 설정 위치: RangeDatePickerWithButtons 컴포넌트에 직접 전달
  • 예시: 시작일이 종료일보다 늦거나, 선택한 기간이 1개월을 초과하는 경우

onValidationError (DatePicker 레벨)

  • 용도: 개별 DatePicker의 minDate/maxDate 범위 검증
  • 검증 항목:
    • minDate: 최소 날짜보다 이전 날짜 선택 시
    • maxDate: 최대 날짜보다 이후 날짜 선택 시
  • 설정 위치: startDateOptions와 endDateOptions에 각각 설정
  • 예시: 시작일이 2025-01-01보다 이전이거나, 2025-12-31보다 이후인 경우

요약

구분onDateValidationonValidationError
레벨RangeDatePickerDatePicker (개별)

검증 대상시작일 ↔ 종료일 관계minDate/maxDate 범위
검증 항목

• period: validationOption.setting에서 설정한 기간 제한 초과
• reset: 날짜 초기화
• null: 유효성 검사 통과

• minDate: 최소 날짜보다 이전 날짜 선택 시

콜백 파라미터{ type: "start" | "end"; errorType: "reset" | "period" | "overlap" | null; newDate: string; currentDate: string; }{ date: Date; minDate?: any; maxDate?: any; violations: Array<'minDate' | 'maxDate'>; previousDate?: Date; }
설정 위치RangeDatePickerWithButtons propsstartDateOptions / endDateOptions
예시시작일이 종료일보다 늦거나, 선택한 기간이 1개월을 초과하는 경우시작일이 2025-01-01보다 이전이거나, 2025-12-31보다 이후인 경우

사용 예시

<RangeDatePickerWithButtons
  // RangeDatePicker 레벨 검증: 시작일과 종료일 간의 관계 검증
  onDateValidation={({ type, errorType, newDate, currentDate }) => {
    if (errorType === 'period') {
      console.log('기간 제한 초과');
    } else if (errorType === 'overlap') {
      console.log('날짜 겹침 오류');
    }
  }}
  validationOption={{
    setting: { unit: 'months', period: 1 }, // 최대 1개월 제한
  }}
  startDateOptions={{
    // DatePicker 레벨 검증: 시작일의 minDate/maxDate 범위 검증
    onValidationError: (error) => {
      const { violations } = error;
      if (violations.includes('minDate')) {
        console.log('최소 날짜 오류');
      } else if (violations.includes('maxDate')) {
        console.log('최대 날짜 오류');
      }
    },
    datePickerOptions: {
      minDate: '2025-01-01',
      maxDate: '2025-12-31',
    },
  }}
  endDateOptions={{
    // DatePicker 레벨 검증: 종료일의 minDate/maxDate 범위 검증
    onValidationError: (error) => {
      const { violations } = error;
      if (violations.includes('minDate')) {
        console.log('최소 날짜 오류');
      } else if (violations.includes('maxDate')) {
        console.log('최대 날짜 오류');
      }
    },
    datePickerOptions: {
      minDate: '2025-01-01',
      maxDate: '2025-12-31',
    },
  }}
/>

onValidationError - minDate/maxDate 범위 validation 처리​

시작일과 종료일 각각의 DatePicker에 onValidationError 콜백을 설정하여 minDate/maxDate 범위를 벗어난 날짜에 대한 처리를 외부에서 할 수 있습니다.

주의사항​

  • currentButtonId와 setCurrentButtonId는 반드시 함께 사용해야 합니다.
  • periodKeys 배열에는 사용하고자 하는 기간 버튼의 ID들을 포함해야 합니다.
  • 날짜를 직접 선택하면 자동으로 currentButtonId가 'NONE'으로 변경됩니다.
  • 기간 버튼 클릭 시 자동으로 해당 기간의 시작일과 종료일이 설정됩니다.
  • useYesterdayAsEndDate가 true일 때, 종료일만 어제 날짜로 설정되며 시작일은 기간 버튼에 따라 계산됩니다.
  • 'ENTIRE' 버튼을 클릭하면 시작일과 종료일이 모두 빈 문자열('')로 설정됩니다.

메서드​

getDates()

선택된 날짜들을 배열로 반환합니다. 첫 번째 요소는 시작일, 두 번째 요소는 종료일입니다.

const dates = datePicker.getDates();
// 예: ['2025-07-01', '2025-07-31']
// 날짜가 선택되지 않은 경우 빈 문자열('')이 반환됩니다.

setDate(dates: string[])

RangeDatePicker의 날짜를 설정합니다.

// 범위 날짜 설정
datePicker.setDate(['2025-07-01', '2025-07-31']);

// 빈 값으로 설정 (placeholder 표시)
datePicker.setDate(['', '']);

옵션​

onValidationError

날짜 유효성 검사 오류 발생 시 호출되는 콜백 함수입니다.

const datePicker = new ncua.DatePicker(wrapper, {
  // ... 기타 옵션
  onValidationError: (error) => {
    const { index, date, minDate, maxDate, violations, previousDate } = error;
    console.log('오류 발생:', violations); // ['minDate'] 또는 ['maxDate']
    console.log('이전 날짜:', previousDate);
    console.log('현재 날짜:', date);

    // 필요시 이전 값으로 복원
    if (previousDate) {
      const currentDates = datePicker.getDates();
      currentDates[index] = previousDate;
      datePicker.setDate(currentDates);
    }
  },
});
